Understanding Semantic HTML

What is Semantic HTML?

Semantic HTML introduces meaning to the web page rather than just presentation. It uses HTML tags that tell something about the content they contain.

For example, tags like <header>, <footer>, <article>, and <section> are semantic because they clearly define their purpose and the type of content they should contain.

Benefits of Semantic HTML

Using semantic elements has several benefits:

  • Accessibility: Helps screen readers and other assistive technologies understand the structure of your content.
  • SEO: Search engines give better rankings to pages that use semantic markup.
  • Maintainability: Makes code easier to read and maintain for developers.
  • Interoperability: Helps different systems understand the structure of your content.

Common Semantic Elements

HTML5 introduced many new semantic elements including:

  • <header> - Represents introductory content
  • <nav> - Represents navigation links
  • <main> - Represents the main content of the document
  • <article> - Represents self-contained composition
  • <section> - Represents a standalone section
  • <aside> - Represents indirectly related content
  • <footer> - Represents footer for its nearest section
